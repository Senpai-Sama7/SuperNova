# Curriculum Lesson Map (Android Instructor-Led Curriculum v1.0)

Use this as a fast map from engineering tasks to the slide decks. If the zip is available, prefer
extracting slide text for the specific lesson and citing slide numbers.

Source zip (typical): `/home/donovan/Downloads/Android-Instructor-Led-Curriculum-v1.0.zip`

## Lessons

- Lesson 0: Introduction Android Development with Kotlin
  - Use for: tooling setup, Android Studio orientation, course framing.
- Lesson 1: Kotlin basics
  - Use for: Kotlin syntax refresh, null safety, collections, control flow.
- Lesson 2: Functions
  - Use for: function design, lambdas, higher-order functions; functional patterns in Kotlin.
- Lesson 3: Classes and objects
  - Use for: Kotlin classes, data classes, interfaces; basic OOP modeling for app code.
- Lesson 4: Build your first Android app
  - Use for: project structure, basic app scaffolding, running/debugging.
- Lesson 5: Layouts
  - Use for: UI layout concepts (note: may be View-based; map cautiously if doing Compose-first).
- Lesson 6: App navigation
  - Use for: navigation patterns, back stack, screen flow design.
- Lesson 7: Activity and Fragment Lifecycles
  - Use for: lifecycle correctness, configuration changes, state/restoration pitfalls.
- Lesson 8: App architecture (UI Layer)
  - Use for: UI state, ViewModel patterns, unidirectional state flow, separation of concerns.
- Lesson 9: App architecture (persistence)
  - Use for: persistence layer, local storage patterns, caching, offline-first discussions.
- Lesson 10: Advanced RecyclerView use cases
  - Use for: complex list UIs (if using Compose, treat as conceptual and translate to Lazy lists).
- Lesson 11: Connect to the internet
  - Use for: networking, API calls, threading/coroutines patterns, error handling.
- Lesson 12: Repository pattern and WorkManager
  - Use for: repositories, background work, scheduling, constraints, reliable execution.
- Lesson 13: App UI Design
  - Use for: UI polish, design principles, consistency, UX review checklists.

## Notes

- Some decks predate Compose; if building Compose UI, extract concepts and map them to Compose idioms.
- Always cite: lesson number + slide number when using the decks to justify a design choice.
